1. Introduction

The Volunteer Management System (VMS) is a responsive web platform enabling volunteers to register, authenticate, explore events, and confirm participation. Administrators configure events, oversee volunteer data, and track engagement. The stack uses HTML, CSS, JavaScript, PHP 8+, and MySQL 8 running on XAMPP.

System Requirements

  • XAMPP (Apache + MySQL)
  • Modern browser (Chrome / Edge)
  • VS Code or similar editor
  • Git (optional but recommended)

Tech Stack

  • Front-end: HTML5, CSS3, JavaScript (ES6)
  • Back-end: PHP with PDO
  • Database: MySQL (InnoDB)
  • Assets: SVG icons, UML diagrams

3. Implementation Roadmap

01

Install XAMPP

Download from apachefriends.org, keep default options, then start Apache + MySQL from the Control Panel.

02

Create Database

Visit http://localhost/phpmyadmin, create database vms, and run the provided database.sql script.

03

Configure Connection

Edit db.php with host/user/password; confirm the PDO success message or catch block.

04

Volunteer Registration

signup.php validates fields and stores password_hash values with default role volunteer.

05

Secure Login

login.php compares user input with hashed credentials via password_verify, then redirects to the dashboard.

06

Session Guard

Include auth.php on any protected page to enforce valid sessions and roles.

07

Volunteer Dashboard

Displays user profile, upcoming events, and join buttons powered by event_registrations.

08

Admin Panel

Admins (role = admin) can approve volunteers, manage roles, and publish events.

09

Run Locally

Place the project in C:/xampp/htdocs/vms, start services, and browse http://localhost/vms/.

10

Logout Flow

logout.php destroys the session and redirects to login.php with a toast.

11

Event Participation

Volunteers click “Join” to insert registrations; duplicates are prevented via UNIQUE constraints.

12

Analytics Snapshot

Dashboard cards track total volunteers, active events, and joined counts.

13

Testing

Test signup/login flows, admin CRUD, and unauthorized access paths before deployment.

14

Optional Upgrades

Implement event reminders, volunteer hours, email notifications, or responsive PWA enhancements.

5. Security & Database Notes

  • Use password_hash/password_verify with Bcrypt.
  • PDO prepared statements shield against SQL injection.
  • Session cookies are httponly with SameSite=Lax.
  • Admins are seeded manually; restrict admin.php by role.
  • Back up the vms database before production deployments.

6. Optional Improvements
